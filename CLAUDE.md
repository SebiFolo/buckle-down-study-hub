# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Buckle Down Study Hub** is a gamified, AI-powered study platform. Students upload notes, get AI-generated summaries, create quizzes/flashcards, and earn XP through a leveling/streak/quest system. Social features include friends, leaderboards, and notifications.

## Commands

```bash
bun run dev          # Vite dev server on localhost:5173
bun run build        # Production build
bun run build:dev    # Dev build
bun run preview      # Preview production build
bun run lint         # ESLint check
bun run format       # Prettier formatting
```

Use `bun` as the package manager (not npm/yarn). Config in `bunfig.toml`.

## Architecture

### Frontend
- **Framework**: TanStack Start (React 19) + TanStack Router (file-based routing in `src/routes/`)
- **Build**: Vite via `@lovable.dev/vite-tanstack-config`
- **Styling**: Tailwind CSS 4 + shadcn/ui (New York style, 30+ Radix UI components in `src/components/ui/`)
- **Data fetching**: TanStack React Query

### Backend
- **Database & Auth**: Supabase (PostgreSQL with Row-Level Security)
- **Edge Functions**: `supabase/functions/` — Deno runtime, each function verifies JWT
- **AI calls**: Made inside edge functions (Claude API), never from the client

### Key Source Directories
| Path | Purpose |
|---|---|
| `src/routes/` | File-based routes; `__root.tsx` is the root layout with auth context + QueryClient |
| `src/components/` | App-level components; `AppShell.tsx` = sidebar (desktop) + bottom nav (mobile) |
| `src/components/ui/` | shadcn/ui components — don't edit these directly |
| `src/lib/` | Domain logic: `leveling.ts`, `xp.ts`, `quests.ts`, `shop.ts`, `friends.ts` |
| `src/integrations/supabase/` | Auto-generated Supabase client & TypeScript types |
| `supabase/functions/` | Edge functions: `generate-study`, `generate-quests`, `generate-summary`, `extract-text`, `award-xp`, `claim-quest`, `shop-buy`, `friends-action` |
| `supabase/migrations/` | SQL schema migrations |

### Routing & Auth
- Public routes: `/`, `/login`, `/signup`, `/forgot-password`, `/reset-password`
- Protected routes: `/dashboard`, `/vault`, `/study`, `/quests`, `/shop`, `/friends`, `/profile`
- Auth state lives in `src/lib/auth-context.tsx` (React Context), consumed throughout the app
- Supabase RLS enforces data access at the database level

### Database Tables (key ones)
`profiles`, `documents`, `flashcard_sets`, `quizzes`, `quests`, `inventory` — all tied to `auth.users`. Profile is auto-created on signup via a Supabase trigger.

### Gamification System
XP/leveling logic is in `src/lib/leveling.ts` and `src/lib/xp.ts`. Level titles go from "Fawn" to "Grand Buck". Daily quests (`src/lib/quests.ts`) and the cosmetics shop (`src/lib/shop.ts`) hook into this system via edge functions (`award-xp`, `claim-quest`, `shop-buy`).

### Responsive Design
Mobile breakpoint is 768px (see `src/hooks/use-mobile.tsx`). `AppShell.tsx` renders a sidebar on desktop and a bottom navigation bar on mobile.

## Code Conventions

- **Path alias**: `@/` maps to `src/` (configured in `tsconfig.json` and `vite.config.ts`)
- **Formatting**: Prettier with `printWidth: 100`, trailing commas — run `bun run format` before committing
- **Types**: `src/integrations/supabase/types.ts` is auto-generated — never edit it manually; regenerate with the Supabase CLI when the schema changes
- **Route tree**: `src/routeTree.gen.ts` is auto-generated by TanStack Router — do not edit manually

## Environment Variables

Required in `.env`:
```
VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
```

Edge functions read `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, and AI API keys from Supabase secrets (not `.env`).
